# Penetration Testing & Social Engineering

## Overview
Penetration testing is a **proactive security practice** used to identify vulnerabilities in systems, networks, and organizations before attackers exploit them.

## Areas of Penetration Testing
Penetration testing can be applied across multiple domains:

- Web applications
- Network and infrastructure
- Internal systems (infected machine simulation)
- Organization-wide testing
- Client-side applications (e.g., C, Java, compiled software)
- Wireless networks
- Mobile applications
- Social engineering
- Phishing and vishing
- Physical security
- Industrial Control Systems (ICS/SCADA)

## Types of Penetration Testing
Based on how much information is provided:

- **Black-box (No knowledge):** No prior information given
- **Grey-box (Partial knowledge):** Limited information provided
- **White-box (Full knowledge):** Full access (source code, diagrams, logs)

> More information generally leads to more effective testing.

## Stolen Laptop Scenario
A lost or stolen laptop can expose sensitive data and credentials.

Risks include:
- Unencrypted drives allowing data extraction
- Logged-in sessions enabling attacker activity
- Malicious hardware (e.g., USB network adapters) intercepting traffic

## Social Engineering
Targets human behavior to manipulate individuals into revealing information or performing actions.

Common psychological triggers:
- Politeness and helpfulness
- Desire to appear knowledgeable
- Trust and kindness
- Fear of authority or consequences

## Social Engineering Scenarios

### 1. Being Helpful
Attackers exploit kindness (e.g., tricking someone into inserting a malicious USB).

### 2. Using Fear
Impersonating authority figures to pressure victims into compliance.

### 3. Reciprocation (Tailgating)
Gaining access by exploiting social norms (e.g., holding doors open).

### 4. Exploiting Curiosity
Dropping infected USB drives to tempt users into plugging them in.

## Phishing
- Conducted via email
- Tricks users into revealing credentials or installing malware
- Used to test user awareness and security controls
- Often executed in campaigns

## Vishing
- Voice-based phishing via phone calls
- Attackers impersonate trusted individuals (e.g., executives)
- Used to extract sensitive information or trigger actions

## Key Takeaways
- Penetration testing simulates real-world attacks to uncover weaknesses
- Human factors are a major security risk
- Social engineering, phishing, and vishing are highly effective attack methods
- Security must address both technical and human vulnerabilities
